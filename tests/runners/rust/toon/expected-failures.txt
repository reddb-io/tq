# Official TOON spec fixtures this crate does not yet satisfy.
# Format: <decode|encode>/<fixture-file>.json::<test name>
#
# Fixtures are read live from the `vendor/toon-spec` submodule, so the corpus
# tracks upstream instead of drifting from a vendored copy.
#
# A decode case counts as passing only when all three hold: it parses, the
# decoded value equals the fixture's `expected` JSON, and our own canonical
# output decodes back to that same value. Parsing without an error is not
# enough — returning the wrong value silently is the failure mode that matters
# most, and it is invisible to an is_ok() check.
#
# This ledger is a ratchet, not a target: entries may only be removed.
